# Mnsx-KrakenPool
> 基于纯 C++11 标准手写的高性能线程池，摒弃 pthread，利用 std::future 与可变参数模板实现安全的异步任务调度，为后端项目提供强大的并发“心脏”

## 项目简介

**Mnsx-KrakenPool** 是基于 C++11 标准开发的高效线程池组件，本项目完全脱离传统的 POSIX 线程库，深入使用 `std::thread` 管理线程声明周期，结合 `std::mutex` 和 `std::conditon_variable` 实现核心的生产者-消费者模型

本项目是在学习完 C++11 现代开发特性后的练习，后续会成为其他 C++ 后端项目（如数据库连接池、文件传输系统）的并发核心

## 核心特性

* **性能优化**：全链路支持 **Move 语义**与**完美转发**，利用 `const_cast` 压榨 `priority_queue` 的移动效率，杜绝冗余拷贝
* **维度提升**：引入**延时任务调度**，支持 `enqueue_after` 和 `enqueue_at` 接口，利用 `std::chrono` 实现毫秒级精度的定时执行
* **工业级防护**：内置有界队列，从源头切断 OOM 风险
* **灵活拒绝策略**：支持 **Abort** 、**CallerRuns** 、**Discard** 以及自旋思想的 `DiscardOldest`
* **零依赖继承**：仅需头文件即可嵌入工程，兼容 C++11 及以上所有主流编译器

## 技术深度

项目采用三层API设计，确保代码的高度复用与执行效率

* `enqueue()`：立即投递
* `enqueue_after()`：相对时间延时投递
* `enqueue_at()`：绝对时间点精准投递

项目采用策略模式提供多种拒绝策略，并且允许自定义拒绝策略器

| 策略              | 行为描述                                                     | 适用场景                               |
| ----------------- | ------------------------------------------------------------ | -------------------------------------- |
| **Abort**         | 默认策略，直接抛出 `runtime_error` 异常                      | 系统**对任务丢失极度敏感**，需上层感知 |
| **CallerRuns**    | 将任务退回给主线程直接执行                                   | **实现自然限流**，保护系统不崩溃       |
| **Discard**       | 默默丢失任务，不产生任何反馈，但是如果获取被丢弃任务的结果，会产生 `Broken promise` 异常 | 适用于**日志采集或非核心数据**         |
| **DiscardOldest** | **使用乐观锁**，踢出队列头部的最老任务，腾位置给新任务，但是如果获取被丢弃任务的结果，会产生 `Broken promise`异常，**暂时被删除** | **实时性要求高**的场景                 |

## 开发路线图

项目采用分阶段迭代开发，目前处于第一阶段

* **[ X ]** **Stage 1：基础骨架**
  * 构建核心 Worker 线程组与任务调度循环
  * 基于条件变量的唤醒机制
* **[ X ]** **Stage 2：泛型支持和异步返回**
  * 引入可变参数模板与完美转发
  * 支持 `std::future` 跨线程获取执行结果
* **[ X ]** **Stage 3：架构解耦和性能压缩**
  * 封装独立的线程安全容器 `SafeQueue`
  * 引入 `std::atomic<bool>` 实现轻量级状态控制，重构 CMake 体系
* **[ X ]** **Stage 4：稳定性增强（过载保护）**
  * **有界队列支持**：防止任务无限堆积导致 OOM
  * **拒绝策略模式**：内置 `Abort` 、`CallerRuns` 、`Discard` 等多种策略
  * **异常处理机制**：优化 `Broken Promise` 等并发异常的规避逻辑
* **[ X ]** **Stage 5：进阶调度特性**
  * **时空调度**：引入 `std::priority_queue` 实现优先级/定时任务
  * **动态唤醒**：工作线程 `wait_until` 动态唤醒机制

* **[   ] Stage 6：深度背压优化**

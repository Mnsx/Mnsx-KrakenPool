# 🐙 Mnsx-KrakenPool
> 基于纯 C++11 标准手写的高性能线程池，摒弃 pthread，利用 std::future 与可变参数模板实现安全的异步任务调度，为后端项目提供强大的并发“心脏”

## 📌 项目简介

**Mnsx-KrakenPool** 是基于 C++11 标准开发的高效线程池组件，本项目完全脱离传统的 POSIX 线程库，深入使用 `std::thread` 管理线程声明周期，结合 `std::mutex` 和 `std::conditon_variable` 实现核心的生产者-消费者模型

本项目是在学习完 C++11 现代开发特性后的练习，后续会成为其他 C++ 后端项目（如数据库连接池、文件传输系统）的并发核心

## 🚀 核心特性

* **纯 C++11 实现**：无任何第三方依赖，极简且易于集成
* **RAII 资源管理**：通过构造函数分配资源，析构函数阻塞等待工作线程（ Join ），杜绝内存泄漏和死锁
* **防御性编程**：严格的生命周期状态校验，禁止拷贝和赋值操作，explicit 修饰构造函数杜绝隐式类型转换

本项目使用 CMake 构建，可以通过以下命令运行基础测试 Demo：

```bash

cmake ..

```

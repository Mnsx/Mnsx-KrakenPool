# Mnsx-KrakenPool
> 基于纯 C++11 标准手写的高性能线程池，摒弃 pthread，利用 std::future 与可变参数模板实现安全的异步任务调度，为后端项目提供强大的并发“心脏”

## 项目简介

**Mnsx-KrakenPool** 是基于 C++11 标准开发的高效线程池组件，本项目完全脱离传统的 POSIX 线程库，深入使用 `std::thread` 管理线程声明周期，结合 `std::mutex` 和 `std::conditon_variable` 实现核心的生产者-消费者模型

本项目是在学习完 C++11 现代开发特性后的练习，后续会成为其他 C++ 后端项目（如数据库连接池、文件传输系统）的并发核心

## 核心特性

* **纯 C++11 核心**：零第三方依赖，极致轻量，一键集成至任何现代工程
* **RAII 资源管理**：通过构造函数自动分配资源，析构函数优雅阻塞（Join），彻底杜绝僵尸线程与内存泄漏
* **防御性编程**：强制 `explicit` 构造，禁用拷贝与赋值逻辑，确保组件在复杂系统中的唯一性与安全性
* **泛型任务支持**：支持提交任意签名的函数、仿函数或 Lambda 表达式
* **异步结果追踪**：结合 `std::packaged_task` 与 `std::future`，实现非阻塞的任务提交与结果回填
* **解耦策略模式**：**[New]** 引入基于 `std::function` 的自定义拒绝策略，轻松应对流量洪峰
* **有界/无界混合队列**：**[New]** 内置支持容量限制的 `SafeQueue`，提供保护机制
* **原子级状态管理**：利用 `std::atomic` 优化内部标志位，压榨微秒级的上下文切换开销

## 拒绝策略

| 策略              | 行为描述                                                     | 适用场景                               |
| ----------------- | ------------------------------------------------------------ | -------------------------------------- |
| **Abort**         | 默认策略，直接抛出 `runtime_error` 异常                      | 系统**对任务丢失极度敏感**，需上层感知 |
| **CallerRuns**    | 将任务退回给主线程直接执行                                   | **实现自然限流**，保护系统不崩溃       |
| **Discard**       | 默默丢失任务，不产生任何反馈，但是如果获取被丢弃任务的结果，会产生 `Broken promise` 异常 | 适用于**日志采集或非核心数据**         |
| **DiscardOldest** | **使用乐观锁**，踢出队列头部的最老任务，腾位置给新任务，但是如果获取被丢弃任务的结果，会产生 `Broken promise`异常 | **实时性要求高**的场景                 |

```bash
mkdir build && cd build
cmake ..
make
./KrakenDemo
```
